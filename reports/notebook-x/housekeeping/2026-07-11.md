The `Data Center` project relies on the `_index-public.json` contract. Currently, there is no server-side validation ensuring that a notebook's `dataQuality` field is restricted to the expected values (`none`, `partial`, `complete`).

**Action:** Add a validation helper in `notebook_backend.py` to prevent malformed index updates.

```python
# Add to notebook_backend.py
VALID_QUALITY_LEVELS = ["none", "partial", "complete"]

def validate_notebook_metadata(metadata):
    if "dataQuality" in metadata and metadata["dataQuality"] not in VALID_QUALITY_LEVELS:
        raise ValueError(f"Invalid dataQuality: {metadata['dataQuality']}. Must be one of {VALID_QUALITY_LEVELS}")
    return True
```
