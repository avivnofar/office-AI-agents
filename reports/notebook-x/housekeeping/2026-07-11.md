### 1. Implement `dataQuality` Validation in `data_safety.py`
Currently, the `dataQuality` field is critical for the Data Center worker. We must ensure that no notebook is marked as `complete` or `partial` unless it meets minimum content requirements (e.g., non-empty files).

**Action:** Add a validation check to `data_safety.py`.

```python
# Add to data_safety.py
def validate_notebook_quality(notebook_data):
    """Ensures notebooks marked as 'complete' or 'partial' have sufficient content."""
    quality = notebook_data.get("dataQuality", "draft")
    if quality in ["complete", "partial"]:
        files = notebook_data.get("files", [])
        if not files or len(files) < 2:
            raise ValueError(f"Notebook quality '{quality}' requires at least 2 files.")
    return True
```

```python
# In notebook_backend.py, inside the function that updates _index-public.json
def sanitize_public_index(notebooks_list):
    required_keys = ["id", "name", "domain", "dataQuality", "summary", "githubRawUrl"]
    sanitized = []
    for nb in notebooks_list:
        # Ensure only public-facing fields are included
        entry = {k: nb.get(k, "N/A") for k in required_keys}
        # Enforce strict dataQuality enum
        if entry["dataQuality"] not in ["draft", "partial", "complete"]:
            entry["dataQuality"] = "draft"
        sanitized.append(entry)
    return sanitized
```
